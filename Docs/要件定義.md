# 就活用ポートフォリオサイト 要件定義

## プロジェクト概要

ハッカソン・イベント・インターン参加経験をまとめた就活用ポートフォリオサイト。
各経験はnoteに詳細記事として書き、このサイトではサマリー（数行）とnoteへのリンクを掲載する。

---

## 目的

- 採用担当者・面接官にエンジニアとしての活動実績を伝える
- noteの詳細記事への導線を作る

## ターゲットユーザー

- 国内企業の採用担当者・エンジニア採用の面接官

---

## 掲載コンテンツ

| カテゴリ | 件数 |
|----------|------|
| ハッカソン | 3件 |
| agile PBL（登壇・参加イベント） | 2件 |
| インターン | 1件 |
| **合計** | **6件** |

---

## 機能要件

### 必須機能

| 機能 | 詳細 |
|------|------|
| 自己紹介セクション | 名前・所属・スキル一覧・一言コメントを表示 |
| 経験一覧セクション | 6件の経験をカード形式で一覧表示 |
| 経験カード | 各経験の詳細情報を表示（下記「カードの掲載情報」参照） |
| noteリンク | 各カードにnote記事へのリンクボタンを設置 |
| GitHubリンク | ヘッダーまたはフッターにGitHubプロフィールへのリンクを設置 |
| レスポンシブ対応 | PC・スマホどちらでも見やすい表示 |

### 経験カードの掲載情報

一般的な就活ポートフォリオでは以下が標準的：

| 項目 | 例 |
|------|----|
| イベント名 | 〇〇ハッカソン2025 |
| 開催年月 | 2025年8月 |
| カテゴリタグ | `ハッカソン` / `インターン` / `イベント` |
| 使用技術タグ | `Flutter` `Dart` `Go` など |
| 役割 | フロントエンド / バックエンド / チームリーダー など |
| チーム人数 | 〇人チーム |
| 受賞歴 | ある場合のみ表示（例: 最優秀賞） |
| サマリー | 3〜5行の概要文 |
| noteリンクボタン | 「詳しく読む →」など |

---

## 非機能要件

| 項目 | 要件 |
|------|------|
| 公開環境 | Vercel（無料プラン） |
| 言語 | 日本語 |
| パフォーマンス | 初回表示3秒以内（静的サイトのため問題なし） |
| 保守性 | 経験を追加する際にコードの変更が最小限であること |

---

## 技術スタック

| 項目 | 採用技術 | 理由 |
|------|----------|------|
| 言語 | TypeScript | 型安全・就活アピールになる |
| フレームワーク | Next.js (App Router) | Vercelとの相性最良、SSGで高速 |
| スタイリング | Tailwind CSS | シンプルモダンなUIを素早く作れる |
| デプロイ | Vercel | GitHub連携で自動デプロイ、無料 |

---

## デザイン方針

- **シンプルモダン**: 白背景ベース、アクセントカラー1色
- カード型レイアウト（6件 → PCは2〜3列グリッド、スマホは1列）
- フォント: システムフォント（日本語）で軽量に
- アニメーション: hover時のみ控えめに

---

## 画面構成

```
/ (トップページ・1ページ完結)
├── ヘッダー（サイト名 + GitHubリンク）
├── 自己紹介セクション
│   ├── 名前・所属
│   ├── 一言コメント
│   └── 技術スタックタグ一覧
├── 経験一覧セクション
│   └── 経験カード × 6枚（グリッドレイアウト）
└── フッター（コピーライト）
```

## データ管理方針

経験データはコード内のTypeScriptファイルで管理する（CMSは使わない）。

```typescript
// src/data/experiences.ts
export type Experience = {
  title: string;
  date: string;         // 例:"2025年8月"
  category: "ハッカソン" | "イベント" | "インターン";
  techs: string[];
  role: string;
  teamSize: number;
  award?: string;       // 受賞した場合のみ
  summary: string;
  noteUrl: string;
};
```
